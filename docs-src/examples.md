# Примеры программ на Brainfix

В данном разделе собраны готовые практические примеры программ, написанных на языке Brainfix. Они наглядно демонстрируют, как использовать синтаксис языка для решения реальных задач, как работают особенности ввода/вывода, массивов и оптимизации, а также как обходить архитектурные ограничения целевой платформы Brainfuck.

Каждый пример снабжён подробным разбором логики и комментариями к коду, чтобы вы могли детальнее разобраться в тонкостях написания эффективных программ.


## Таблица умножения

Этот пример демонстрирует работу с вложенными циклами (`while` и `for`), использование префиксного инкремента в условии, а также вывод сложных составных выражений на экран. Программа выводит классическую таблицу умножения чисел от 2 до 9.

### Исходный код

```brainfix
# @title: Умножение

byte i = 1;

while (++i < 10) {
    for (byte j = 2; j < 10; j++)
        out i, ' * ', j, ' = ', i * j, eol;

    out eol;
}
```

### Разбор логики работы

1. **Инициализация счётчика строк (`byte i = 1;`)**: Создаётся переменная внешнего цикла. Мы начинаем с единицы, так как инкремент происходит прямо в условии до начала первой итерации.
2. **Внешний цикл (`while (++i < 10)`)**: Здесь используется префиксный инкремент `++i`. Это означает, что при первом входе в цикл значение `i` сначала увеличится до `2`, а затем проверится условие `2 < 10`. Цикл будет выполняться до тех пор, пока `i` не станет равным `10`.
3. **Внутренний цикл (`for (byte j = 2; j < 10; j++)`)**: Для каждой строки `i` запускается цикл по переменной `j` от 2 до 9. Обратите внимание, что фигурные скобки `{}` здесь опущены, так как внутри выполняется ровно одна команда `out`.
4. **Комбинированный вывод (`out ...`)**: Команда `out` за один вызов принимает цепочку из переменных, символьных литералов и математического выражения `i * j`. В конце строки используется ключевое слово `eol` для перевода каретки.
5. **Разделение блоков (`out eol;`)**: После того как внутренний цикл выведет целый столбец для текущего числа `i`, внешний цикл печатает пустую строку `eol` для визуального разделения таблицы.


## Транслитерация строки

Этот комплексный пример демонстрирует работу с двумерными массивами строк, ввод строк, арифметику над кодами символов (ASCII/Windows-1251) и перевод букв в верхний регистр. Программа заменяет русские буквы на их латинские аналоги.

### Исходный код

```brainfix
# @title: Транслит


char map[][] = [
  'a', 'b', 'v', 'g', 'd', 'e', 'zh', 'z', 'i', 'y',
  'k', 'l', 'm', 'n', 'o', 'p', 'r', 's', 't', 'u',
  'f', 'h', 'c', 'ch', 'sh', 'shch', '', 'y', '', 'e',
  'yu', 'ya'
];

char letter;

do
{
    in letter;

    if (letter >= 'а' && letter <= 'я')
    {
        out map[letter - 'а'];
    }
    else if (letter >= 'А' && letter <= 'Я')
    {
        for (byte i = 0; i < sizeof map[0]; i++)
        {
            if (map[letter - 'А'][i])
            {
                char ch = map[letter - 'А'][i] - ('a' - 'A');
                out ch;
            }
        }
    }
    else
    {
        out letter;
    }
}
while (letter != eol);

```

### Разбор логики работы

1. **Создание таблицы замен (`char map[][]`)**: Объявляется двумерный массив строк. Компилятор автоматически определит его размер как `32x4` (32 буквы русского алфавита, исключая «ё», а максимальная длина латинского эквивалента — 4 символа для `'shch'`). Пустые кавычки `''` заменяют знаки «ъ» и «ь», которые при транслитерации опускаются.
2. **Потоковое чтение (`do { in letter; ... } while (letter != eol);`)**: Цикл `do/while` гарантирует, что программа считает и обработает как минимум один символ. Чтение происходит посимвольно до тех пор, пока не встретится символ переноса строки (`eol`).
3. **Обработка строчных букв (`letter >= 'а' && letter <= 'я'`)**:
    * Вычисляется индекс буквы в массиве путём вычитания кода первой буквы алфавита: `letter - 'а'`.
    * С помощью `out map[...]` вся строка транслита для этой буквы выводится на экран. Благодаря оптимизации многомерных массивов, этот переход по динамическому индексу выполняется быстрее, чем в плоском массиве.
4. **Обработка прописных (заглавных) букв (`letter >= 'А' && letter <= 'Я'`)**:
    * Так как в массиве `map` все строки хранятся в нижнем регистре, заглавные буквы нужно переводить вручную.
    * Цикл `for` обходит элементы строки транслита. Длина строки ограничена размером второй размерности массива (`sizeof map[0]`, то есть 4 итерации).
    * Проверка `if (map[...][i])` отсекает пустые ячейки (нули, которыми компилятор автоматически заполнил короткие строки вроде `'a'`).
    * Математическое выражение `map[...][i] - ('a' - 'A')` вычитает разницу между строчными и прописными буквами в таблице кодировки, переводя текущий символ в верхний регистр. Поскольку явного приведения типов нет, результат операции сначала сохраняется в промежуточную переменную `char ch`, которая затем выводится.
5. **Сторонние символы (`else { out letter; }`)**: Знаки препинания, пробелы, цифры и латинские буквы выводятся обратно в поток без изменений.


## Интерпретатор Brainfuck на Brainfix

Этот пример демонстрирует создание полноценного интерпретатора языка Brainfuck. Программа парсит строку с исходным кодом, эмулирует классическую ленту памяти, управляет её указателем и обрабатывает вложенные циклы `[` и `]`, перемещая внутренний индекс по массиву инструкций.

### Исходный код

```brainfix
# @title: Транслятор

char code[256];
char memory[100];
byte pointer, level;

in code;

for (byte i = 0; i < sizeof code; i++)
{
    char command = code[i];

    if (command == '>')
    {
        pointer++;
    }
    else if (command == '<')
    {
        pointer--;
    }
    else if (command == '+')
    {
        memory[pointer]++;
    }
    else if (command == '-')
    {
        memory[pointer]--;
    }
    else if (command == '.')
    {
        out memory[pointer];
    }
    else if (command == ',')
    {
        in memory[pointer];
    }
    else if (command == '[' && !memory[pointer])
    {
        level++;
        while (level)
        {
            i++;
            if      (code[i] == '[') level++;
            else if (code[i] == ']') level--;
        }
    }
    else if (command == ']' && memory[pointer])
    {
        level++;
        while (level)
        {
            i--;
            if      (code[i] == '[') level--;
            else if (code[i] == ']') level++;
        }
        i--;
    }
}
```

### Разбор логики работы

1. **Выделение памяти под виртуальную машину**:
    * `char code[256]` — буфер для хранения считываемого кода Brainfuck.
    * `char memory[100]` — эмулируемая лента памяти для исполняемой Brainfuck-программы. Изначально вся лента заполнена нулями.
    * `byte pointer` — указатель на текущую ячейку эмулируемой памяти `memory`.
    * `byte level` — счётчик уровня вложенности скобок для корректной обработки циклов.
2. **Чтение исходного кода (`in code;`)**: Команда считывает всю введённую пользователем строку с Brainfuck-кодом в массив `code`.
3. **Главный цикл интерпретатора (`for ... sizeof code`)**: Программа последовательно шагает по массиву `code` индекс за индексом, извлекая текущую команду в переменную `command`.
4. **Обработка простых команд (`>`, `<`, `+`, `-`, `.`, `,`)**:
    * Сдвиги `>` и `<` инкрементируют и декрементируют виртуальный `pointer`.
    * Изменения `+` и `-` модифицируют значение в ячейке `memory[pointer]`.
    * Операции ввода/вывода `.` и `,` напрямую передают данные из виртуальной ленты в реальный поток исполнения.
5. **Эмуляция начала цикла (`[` и текущая ячейка равна 0)**:
    * Если текущая ячейка памяти равна нулю, цикл выполнять не нужно, и интерпретатор должен прыгнуть вперёд к парной закрывающей скобке `]`.
    * Переменная `level` увеличивается до 1. В цикле `while (level)` индекс `i` двигается вперёд по коду. Если по пути встречаются вложенные `[`, `level` растёт, если `]`, то уменьшается. Как только `level` падает до 0, это означает, что парная скобка найдена, и выполнение продолжится со следующей за ней команды.
6. **Эмуляция конца цикла (`]` и текущая ячейка НЕ равна 0)**:
    * Если текущая ячейка памяти не равна нулю, программу нужно вернуть назад к началу цикла (к парной открывающей скобке `[`).
    * Логика поиска аналогична: `level` увеличивается, индекс `i` двигается назад (`i--`), пока счётчик вложенности снова не обнулится.
    * После выхода из `while (level)` указатель `i` будет стоять ровно на символе `[`. Чтобы на следующей итерации главного цикла `for` шаг `i++` не перескочил первую команду внутри цикла, мы принудительно делаем дополнительный откат `i--`.
