# Документация Brainfix

**Brainfix** - это Си-подобный, строго типизированный язык программирования, созданный для разработки программ на Brainfuck. На этой странице собрано полное руководство по синтаксису, которое поможет вам детально изучить язык и начать писать свои программы.

## Первая программа

Классический пример «Hello, world!» в Brainfix состоит всего из одной строки кода:

```brainfix
out 'Hello, world!';
```

Скомпилированная в brainfuck программа содержит 188 инструкций:
```brainfuck
>++++++++[-<+++++++++>]<.>++++[-<+++++++>]<+.+++++++..+++.>++++++[-<----------->]<-.------------.>+++++[-<+++++++++++>]<.>++++[-<++++++>]<.+++.------.--------.>++++++[-<----------->]<-.[-]
```

## Комментарии

Комментарии используются для пояснения логики программы и игнорируются компилятором при сборке кода. Они помогают делать исходный код более читаемым для человека.

В Brainfix поддерживаются только **однострочные комментарии**. Они начинаются с символа решётки `#`. Всё, что написано после этого символа и до конца текущей строки, компилятор считает комментарием.

```brainfix
# Это комментарий, который занимает целую строку
byte speed = 100; # А это комментарий, расположенный после кода

# Символ # внутри строкового литерала не является комментарием:
out "Символ # здесь выведется как обычный текст";
```

## Переменные и типы данных

Названия переменных в Brainfix регистрозависимы. Они могут состоять из латинских букв, цифр, символов `_` и `$`, но не могут начинаться с цифры.

```brainfix
# Правильные имена:
a, b, char1, firstNumber, $a, _B, count, COUNT
```

Brainfix — строго типизированный язык. Тип данных должен быть явно указан для каждой переменной.


В языке всего 3 базовых(скалярных) типа данных:
* `char` - Символ в кодировке `windows-1251`.
* `byte` - Целое число от 0 до 255. От `char` отличается только форматом ввода/вывода.
* `bool` - Булев тип (`true`/`false`).

### Объявление и инициализация
Переменные можно объявлять с одновременным присвоением значения (одиночным или цепочкой):
```brainfix
# Объявление и инициализация
char char_1 = 'a';
byte count = 0, newCount = 30;
bool b = false;

# Множественное присваивание
byte first = second = third = 1;
```

### Объявление без инициализации
Если переменная объявляется без начального значения, память под неё выделяется, но не очищается.
Переменная может содержать случайные данные ("мусор"), оставшиеся в этой ячейке.

```brainfix
# Значения переменных не гарантировано равны нулю
byte n;
char ch1, ch2;
bool isFull; # Значение була может быть больше 1
```

## Литералы

Литералы — это фиксированные значения, которые записываются непосредственно в коде программы. В Brainfix поддерживаются следующие виды литералов:

### Логические литералы
Представлены двумя ключевыми словами для булевых значений:
* `true` — логическая истина. Соответствует единице.
* `false` — логическая ложь. Соответствует нулю.

### Числовые литералы
Записываются в виде целых чисел. Числовые литералы в коде могут превышать максимальное значение типа `byte` (255).

> При присвоении числового литерала переменной типа `byte`, итоговое значение вычисляется **по модулю 256**.

```brainfix
byte a = 5;   # В памяти: 5
byte b = 256; # В памяти: 0 (256 % 256)
byte c = 300; # В памяти: 44 (300 % 256)
```

### Символьные и строковые литералы
В Brainfix как отдельные символы, так и строки могут обрамляться как в **одинарные (`'`), так и в двойные (`"`) кавычки**. Тип литерала определяется автоматически по количеству символов внутри кавычек.

* **Символьный литерал** — это ровно один символ в кавычках. Используется для инициализации типа `char`.
* **Строковый литерал** — это последовательность из двух и более символов в кавычках.

```brainfix
char letter = 'A';    # Символьный литерал
char symbol = "Z";    # Тоже символьный литерал (один символ)

out "Hello", 'Hello'; # Это строковые литералы
```

### Управляющие символы и перенос строки
В Brainfix есть два способа записи перевода строки:
* `eol` — специальное ключевое слово, обозначающее символ переноса строки.
* `\n` — управляющий символ, который можно использовать внутри строковых литералов.

```brainfix
# Оба варианта в данном случае работают одинаково
out 'Hello, World!', eol;
out 'Hello, World!\n';
```




## Области видимости переменных

Область видимости (scope) определяет границы участка программы, внутри которого имя переменной остается известным и может быть использовано.

### Блочная область видимости

В Brainfix область видимости переменных всегда ограничена **фигурными скобками `{}`**. Любая переменная, объявленная внутри блока кода, является локальной для этого блока. Она создается при входе в блок и уничтожается при выходе из него.

```brainfix
{
    byte local_x = 10;
    # Переменная local_x доступна здесь
}
# Здесь переменная local_x больше не существует.
out local_x; # Вызовет ошибку variable 'local_x' not defined
```

### Видимость во вложенных блоках

Внутренние блоки кода имеют доступ ко всем переменным, которые были объявлены во внешних по отношению к ним блоках.

```brainfix
byte global_count = 50;

{
    # Внутренний блок видит переменную из внешнего блока
    byte current = global_count + 10; 
}
```

### Перекрытие (затенение) переменных

Если во внутреннем блоке объявить переменную с точно таким же именем, как и во внешнем блоке, то внутренняя переменная **перекроет (затемнит)** внешнюю.

Внутри этого блока все обращения по имени будут идти к новой локальной переменной. Внешняя переменная при этом не изменяется и снова станет доступна, как только выполнение программы выйдет из внутреннего блока.

```brainfix
byte value = 10;

{
    byte value = 255; # Перекрывает внешнюю переменную value
    # Здесь value равен 255
}

# Здесь внутренний блок завершился, value снова равен 10
```

### Особая область видимости в циклах

Для цикла `for` действует дополнительное правило: переменные, объявленные внутри круглых скобок в секции инициализации `for(здесь;;) `, доступны только внутри этого цикла (включая его тело) и не видны за его пределами.

```brainfix
for (byte i = 0; i < 10; i = i + 1) {
    # Переменная i доступна внутри этого блока
    byte result = i * 2;
}
out i; # Ошибка! Переменная i здесь недоступна
```


## Операторы

Операторы в Brainfix используются для выполнения арифметических вычислений, логических операций, сравнения значений и присваивания.

### Арифметические операторы

Базовые математические операции:
* `+` — сложение.
* `-` — вычитание.
* `*` — умножение.
* `/` — деление.
* `%` — остаток от деления.

### Инкремент и декремент

Используются для увеличения или уменьшения значения переменной на единицу. Поддерживаются в двух формах:
* **Префиксная (`++a`, `--a`)** — сначала изменяет значение переменной, затем возвращает результат.
* **Постфиксная (`a++`, `a--`)** — сначала возвращает текущее значение переменной, а затем изменяет его.

### Операторы сравнения

Используются для сопоставления двух значений. Результатом операции всегда является логическое значение типа `bool` (`true` или `false`).

* `<` — меньше.
* `>` — больше.
* `<=` — меньше или равно.
* `>=` — больше или равно.
* `==` (или `===`) — равно.
* `!=` (или `!==`, `<>`) — не равно.

```brainfix
byte x = 5;
byte y = 10;
bool result1 = (x < y);   # true
bool result2 = (x == y);  # false
bool result3 = (x <> y);  # true (алиас для !=)
```

### Логические операторы

Применяются к выражениям типа `bool` для построения сложных логических условий:
* `&&` — логическое И (конъюнкция).
* `||` — логическое ИЛИ (дизъюнкция).
* `!` — логическое НЕ (отрицание).

### Операторы присваивания и оптимизация

Помимо базового присваивания, язык поддерживает комбинированные операторы, которые совмещают арифметическое действие с записью результата:

* `=` — базовое присваивание.
* `+=` — сложение с присваиванием.
* `-=` — вычитание с присваиванием.
* `*=` — умножение с присваиванием.
* `/=` — деление с присваиванием.
* `%=` — остаток от деления с присваиванием.

## Приведение типов

В языке Brainfix действует гибкая система автоматического (неявного) преобразования между всеми тремя скалярными типами (`byte`, `char`, `bool`).

### Автоматическое приведение при присваивании

Вы можете свободно присваивать значение любого скалярного типа переменной другого типа. Компилятор выполнит преобразование автоматически по следующим правилам:

* **Число и Символ (`byte` <-> `char`)**: Присваивание происходит на основе кода символа в кодировке `windows-1251`.
* **Логический тип в Число/Символ (`bool` -> `byte`/`char`)**: Значение `true` преобразуется в число `1` (или символ с кодом 1), а `false` — в `0`.
* **Число/Символ в Логический тип (`byte`/`char` -> `bool`)**: Любое ненулевое значение (не равное `0`) преобразуется в `true`. Нулевое значение становится `false`.

```brainfix
# byte <-> char
char ch = 'A';    # Символ 'A' имеет код 65
byte b = ch;      # b станет равно 65
char ch2 = b + 1; # ch2 станет равен 'B'

# bool -> byte
bool flag = true;
byte num = flag;  # num станет равно 1

# byte -> bool
byte count = 42;
bool isExist = count; # count не равен 0, поэтому isExist станет true
```

### Приведение типов в выражениях

Арифметические операторы (`+`, `-`, `*`, `/`, `%`) **всегда возвращают числовой результат (тип `byte`)**, независимо от типов операндов.

Если в выражении участвуют переменные типа `char` или `bool`, они будут автоматически приведены к их числовому эквиваленту перед выполнением математической операции.

```brainfix
char letter1 = 'A'; # код 65
char letter2 = 'B'; # код 66

# Результатом сложения двух символов будет число 131 (тип byte)
byte result = letter1 + letter2; 

bool active = 5; # преобразуется в 1
byte total = letter1 + active; # 65 + 1 = 66
```

> В Brainfix отсутствует синтаксис явного приведения типов (вроде `(byte)x`). Все преобразования происходят исключительно в автоматическом режиме на основе контекста использования переменной.


## Ввод и вывод

Для взаимодействия с пользователем в Brainfix используются всего две команды: `in` (для чтения данных) и `out` (для вывода данных). Обе команды поддерживают перечисление нескольких аргументов через запятую.

Формат обработки данных полностью зависит от типа выражения или литерала, переданного в команду.

### Вывод данных (`out`)

Команда `out` выводит значения в поток вывода. Её поведение зависит от типа аргумента:

* **Литералы**: Выводятся "как есть". Числовые литералы не подвергаются ограничению типа `byte`. Например, `out 257;` напечатает на экране ровно `257`, без обрезания по модулю 256.
* **Переменные типа `char`**: Выводятся как одиночный символ.
* **Переменные типа `byte`**: Выводятся как числовое значение.
  > **Важно:** Для вывода значений типа `byte` используется сложный алгоритм, включающий до трех операций деления. Избегайте циклического или слишком частого вывода числовых выражений.
* **Переменные типа `bool`**: Выводятся в виде цифры `1` (для `true`) или `0` (для `false`).

```brainfix
byte num = 65;
char letter = 65;
bool flag = true;

out "Результаты: ", num, ' ', letter, ' ', flag; 
# Выведет: Результаты: 65 A 1

# Можно также выводить выражения
out 'num * 2 = ', num * 2;
# Выведет: num * 2 = 130
```

### Особенности вывода выражений

Поскольку в Brainfix формат вывода полностью завязан на возвращаемый тип выражения, а явного приведения типов (cast) нет, в некоторых ситуациях стандартный `out` может вывести не то, что вы ожидали.

Например:
1. Вы хотите вывести число типа `byte` как соответствующий ему ASCII-символ.
2. Вы хотите сместить символ по таблице кодировки (например, `char1 + 1`), но результат этой арифметической операции автоматически становится типом `byte` и выводится как число.

Чтобы обойти это ограничение, используйте промежуточные переменные нужного типа.

```brainfix
char char1 = 'A'; # код 65

# ПРОБЛЕМА: Арифметическая операция возвращает byte.
# Код ниже выведет число "66", а не символ "B"
out char1 + 1; 

# РЕШЕНИЕ: Заводим временную переменную нужного типа
char nextChar = char1 + 1; 
out nextChar; # Выведет символ "B"

# Аналогично для вывода числа в виде символа:
byte code = 33;
char symbol = code;
out symbol; # Выведет символ "!"
```

### Ввод данных (`in`)

Команда `in` считывает данные из стандартного потока ввода. Поведение зависит от типа переменной, в которую записывается значение:

* **Литералы (Пропуск ввода)**: Если в команду `in` передать любой литерал (например, `in 'dummy';`), программа прочитает ровно один символ из ввода и просто **выбросит его**. Это полезно для пропуска ненужных символов (разделителей) или для реализации логики "Нажмите любую клавишу для продолжения".
* **Переменные типа `char`**: Самый быстрый и простой ввод. Считывает из потока один символ и записывает его код в переменную.
* **Переменные типа `byte`**: Считывает все символы до первого пробела или нажатия `Enter`(включая их). Полученная строка преобразуется в число.
  Введенное число может быть больше 255. Тогда в переменную запишется значение по модулю 256.
  Если пользователь введет не цифры, а буквы, в переменную запишется непредсказуемый "мусор". Такое поведение можно использовать для генерации случайных чисел (сидов рандомизации).
* **Переменные типа `bool`**: Считывает один символ. Если пользователь вводит `0` (код 48), значение переменной становится равно `false`. Во всех остальных случаях - `true`.

```brainfix
char ch;
byte age;
bool agreement;

# Ожидание ввода
in ch, ' ', age, agreement;
# Возможные форматы ввода:
# A 25 1
# A
# 25
# 1
# A*25 1

# Пропуск символа
out 'Нажмите любую кнопку';
in 'dummy';

# Если нужно получить псевдослучайное число
byte seed;
out 'Введите любое слово';
in seed; # В seed запишется псевдослучайное число
```
